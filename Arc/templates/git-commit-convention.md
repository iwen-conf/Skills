## Engineering Habits: Git Commit Conventions

All Git commit messages in this repository must adhere to the following convention:
- **Format:** `<type>: <description>`
- **Types:** `feat` (new feature), `fix` (bug fix), `refactor` (code change that neither fixes a bug nor adds a feature), `docs` (documentation only), `chore` (build process or auxiliary tools).
- **Description:** One concise sentence summarizing the change. No filler words. Focus on *what* changed and *where*. Keep it professional.
- **Example:** `fix: resolve race condition in context module loading`
