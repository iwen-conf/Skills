# Fusion Guide

Arc 只保留 `arc:*` 软件工程生命周期 Skill。组合原则很简单：

- 任务编排、记忆和协作交给 `aitask`。
- Arc 提供定义、澄清、飞书项目空间、交付、前端工程、修复、审计七类工作方法。
- 飞书项目空间不按项目大小自动创建：只有用户明确要求创建/连接项目空间，或项目已有 `.lark.json`，才由 `arc:docs` 维护飞书资源。
- 后续搜索资料、新增资料、决策、交付证据和会议/审计产物属于已有 `.lark.json` 的项目时，必须通过 `arc:docs` 写入对应飞书资源。
- 垂直领域能力交给对应 Skill，不在 Arc 内重复建设；Arc 只记录必要链接和生命周期状态。
