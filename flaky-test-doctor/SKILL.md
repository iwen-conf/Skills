# flaky-test-doctor

## Overview

将随机失败转成可复现问题，再对症修复与加固。

## When to Use

- CI 测试偶发失败
- 本地通过但流水线失败
- 重试后通过的“假稳定”测试

## Outputs

- 不稳定根因分类（时序/并发/外依赖/环境）
- 最小复现路径
- 修复方案与反脆弱策略
- 稳定性回归验证记录

## Workflow

1. 收集失败样本并分组（时间、环境、依赖）
2. 构建最小可复现路径
3. 按根因类型实施修复
4. 进行稳定性回归验证并记录波动
