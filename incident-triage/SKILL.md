# incident-triage

## Overview

将故障处理流程化，先恢复服务，再定位根因。

## When to Use

- 线上故障或严重告警触发
- 多系统联动异常
- 需要快速组织跨团队响应

## Outputs

- 事件分级与影响范围
- 止血方案与执行记录
- 根因分析结论
- 复盘条目与行动项

## Workflow

1. 快速分级并确认影响范围
2. 优先执行止血与服务恢复
3. 追踪根因并补充证据链
4. 输出复盘与行动项闭环
