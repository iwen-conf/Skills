# CN 软件知识产权可行性评估准则

## 软件著作权评估

- 代码是否达到可提交规模并具备版本边界。
- 软件名称、版本、申请主体是否可统一。
- 操作说明素材是否完整（功能流程、截图、环境）。
- 是否存在权属争议（职务开发、外包、开源协议冲突）。

## 发明专利评估

- 是否明确解决了具体技术问题。
- 是否使用可验证的技术手段（而非纯业务规则）。
- 是否可量化技术效果（性能、稳定性、资源占用）。
- 与现有技术相比是否具备实质差异。

## 风险等级

- `LOW`：材料齐全，技术边界清晰，可直接推进。
- `MEDIUM`：存在材料缺口或术语不统一，需要补充。
- `HIGH`：核心创新点证据不足或客体不稳定。
