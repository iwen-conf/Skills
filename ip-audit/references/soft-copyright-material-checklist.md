# 软著申请材料清单

## 1. 代码材料

### 格式要求
- **纸张**: A4
- **颜色**: 黑白
- **打印**: 单面
- **行数**: ≥50行/页
- **页眉**: 软件全称 + 版本 + 页码（如：智能任务调度系统 V1.0 第1页）
- **页脚**: 著作权人（如：北京某某科技有限公司）

### 页数要求
- **<60页**: 必须全量提交（不能提交首尾样本）
- **≥60页**: 可提交首尾连续样本（各30页）或全量提交
- **线上提交**: 可超60页无需裁剪

### 代码样本选择建议
- 优先选择核心功能模块代码
- 避免第三方库代码
- 删除敏感信息（密钥、密码、内部URL）
- 删除过多注释（保留关键注释）
- 确保代码连续性（不要跳跃式截取）

### 检查清单
- [ ] 代码行数≥50行/页
- [ ] 页眉包含：软件全称 + 版本 + 页码
- [ ] 页脚包含：著作权人
- [ ] 页眉页脚中的软件名称与申请表完全一致
- [ ] 代码已脱敏（无敏感信息）
- [ ] 代码连续（无跳跃）
- [ ] 页数符合要求（<60页全量，≥60页首尾样本或全量）

---

## 2. 说明文档

### 格式要求
- **行数**: ≥35行/页
- **截图命名**: 必须包含软件全称，与申请表、页眉页脚完全一致
- **界面名称**: 与软件名绝对一致

### 内容要求
- 软件功能概述
- 运行环境（操作系统、硬件要求）
- 安装步骤
- 使用说明（主要功能操作流程）
- 界面截图（带标注）
- 故障排除（可选）

### 检查清单
- [ ] 说明文档行数≥35行/页
- [ ] 截图中的软件名称与申请表完全一致
- [ ] 界面名称与软件名完全一致
- [ ] 包含安装、配置、使用全流程说明
- [ ] 截图清晰可辨认
- [ ] 页眉页脚格式与代码材料一致

---

## 3. 申请表

### 必填信息
- **软件全称**: 必须以"软件/系统/平台/APP/插件"结尾
- **软件简称**: 长度不得超过全称
- **版本号**: 与代码材料页眉一致
- **著作权人**: 与代码材料页脚一致
- **开发完成日期**: 不得晚于首次发表日期
- **首次发表日期**: 可选，未发表填"未发表"

### 检查清单
- [ ] 软件全称末尾包含"软件/系统/平台/APP/插件"
- [ ] 软件简称长度≤全称长度
- [ ] 版本号与代码材料页眉一致
- [ ] 著作权人与代码材料页脚一致
- [ ] 著作权人与实名账号姓名/企业名称完全一致
- [ ] 开发完成日期≤首次发表日期

---

## 4. 签章页与声明

### 签章页（独立电子签名页）
- **要求**: 2023年起全量线上办理，需独立电子签名页
- **签名方式**: 在线电子签名（不接受扫描件）
- **签名人**: 著作权人或法定代表人

### 非职务开发保证书（个人申请必需）
- **适用**: 个人申请人
- **内容**: 声明该软件为非职务开发作品
- **签名**: 本人签名 + 日期

### 开源协议声明（如有开源依赖）
- **适用**: 使用了开源组件的软件
- **内容**: 列出开源组件名称、版本、许可证类型
- **说明**: 声明本软件与开源组件的关系（如：使用、修改、集成）

### 检查清单
- [ ] 签章页已准备（电子签名）
- [ ] 个人申请已准备非职务开发保证书
- [ ] 如有开源依赖，已准备开源协议声明
- [ ] 签名人与著作权人一致
- [ ] 实名账号姓名与申请表、页眉页脚完全一致

---

## 5. App 电子版权（可选快线）

### 适用场景
- 需要快速上架应用市场（iOS App Store、Android应用商店）
- 需要1-10个工作日快速出证
- 区块链确权通道

### 所需材料
- 电子版权页（区块链确权证明）
- 软著认证页（传统软著证书）
- App 安装包（APK/IPA）
- App 截图（主要功能界面）

### 办理流程
1. 先申请传统软著（或同时申请）
2. 登录电子版权平台（如：版权家、版权印）
3. 上传 App 安装包 + 截图
4. 区块链存证
5. 获取电子版权页
6. 用于应用市场上架

### 检查清单
- [ ] 已申请或正在申请传统软著
- [ ] App 安装包已准备（APK/IPA）
- [ ] App 主要功能截图已准备
- [ ] 软件名称与软著申请表一致

---

## 6. 命名统一铁律

### 统一要求
所有材料中的软件名称必须**像素级一致**：
- 申请表中的软件全称
- 代码材料页眉中的软件全称
- 代码材料页脚中的著作权人
- 说明文档中的软件名称
- 说明文档截图中的软件名称
- 界面显示的软件名称
- 实名账号姓名/企业名称

### 常见错误
- ❌ 申请表：智能任务调度系统，页眉：任务调度系统
- ❌ 申请表：智能任务调度系统 V1.0，页眉：智能任务调度系统V1.0（空格不一致）
- ❌ 申请表：北京某某科技有限公司，页脚：某某科技
- ❌ 截图：智能任务调度平台，申请表：智能任务调度系统

### 检查清单
- [ ] 申请表、页眉、页脚、截图中的软件名称完全一致
- [ ] 软件全称末尾包含"软件/系统/平台/APP/插件"
- [ ] 版本号格式一致（如：V1.0 或 v1.0，全文统一）
- [ ] 著作权人名称与实名账号完全一致

---

## 7. 无纸化实名前置

### 实名要求
- **账号实名**: 在中国版权保护中心官网完成实名认证
- **姓名/企业名称**: 必须与申请表、页眉页脚完全一致
- **个人申请**: 需上传身份证正反面
- **企业申请**: 需上传营业执照副本

### 电子签名
- **签名方式**: 在线电子签名（不接受扫描件）
- **签名时间**: 提交申请后，系统会发送签名通知
- **签名人**: 著作权人或法定代表人

### 检查清单
- [ ] 已完成账号实名认证
- [ ] 实名账号姓名/企业名称与申请表一致
- [ ] 个人申请已上传身份证
- [ ] 企业申请已上传营业执照
- [ ] 了解电子签名流程

## 9. 常见驳回原因与预防

| 驳回原因 | 预防措施 |
|---------|---------|
| 软件名称不一致 | 使用格式合规检查脚本验证 |
| 代码行数不足 | 确保≥50行/页 |
| 说明文档行数不足 | 确保≥35行/页 |
| 页眉页脚格式错误 | 严格按照模板格式 |
| 实名信息不一致 | 申请前核对账号实名信息 |
| 代码不连续 | 提交前检查代码连续性 |
| 开源协议冲突 | 提前声明开源依赖 |

---

## 10. 办理时间与费用

### 办理时间
- **普通软著**: 30-60个工作日
- **加急软著**: 1-10个工作日（需额外费用）
- **App 电子版权**: 1-10个工作日

### 费用
- **软著申请费**: 免费（官方不收费）
- **加急费**: 根据加急程度，300-1000元不等
- **代理费**: 如委托代理机构，500-2000元不等

### 费减政策
软著申请费本身免费，无需费减。
