## 文档写作规范

### 通用规范

1. **术语一致性**: 同一对象全文使用统一术语,不得随意替换同义词
2. **证据可回溯**: 每个技术描述必须能回溯到项目代码或审查交接文件
3. **量化优先**: 技术效果尽量量化(性能提升X%, 资源降低Y%)
4. **避免模糊表述**: 禁止使用"可能"、"大概"、"一般"、"通常"等模糊词汇
5. **脱敏处理**: 移除商业机密、敏感数据、内部代号

### 软著文档规范

- **面向对象**: 版权局审查员(非技术专家)
- **语言风格**: 清晰、易懂、完整
- **重点**: 功能完整性、操作可行性、文档完备性

### 专利文档规范

- **面向对象**: 专利审查员(技术专家)
- **语言风格**: 精确、严谨、可实施
- **重点**: 技术创新性、技术效果显著性、与现有技术差异性
- **法律要求**: 符合《专利法》第26条"充分公开"原则

## 质量检查清单

### 软著文档检查

- [ ] 软件名称全文一致
- [ ] 版本号明确且与代码一致
- [ ] 功能描述完整(覆盖核心功能)
- [ ] 操作说明可执行(已验证步骤)
- [ ] 代码区段符合版权局要求(3000-5000行)
- [ ] 第三方依赖已标注且许可证兼容

### 专利文档检查

- [ ] 技术问题清晰且量化
- [ ] 技术方案完整且可实施
- [ ] 技术效果显著且有对比数据
- [ ] 与现有技术差异明确
- [ ] 权利要求层次清晰(独立+从属)
- [ ] 附图标注与文字说明对应
- [ ] 术语全文一致
