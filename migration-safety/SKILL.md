# migration-safety

## Overview

围绕“可前进、可回退、可校验”设计迁移流程。

## When to Use

- 表结构变更、索引调整、数据回填
- 高并发系统需要平滑迁移
- 发布窗口有限、回滚要求高

## Outputs

- 前向迁移脚本与回滚脚本
- 数据校验计划（前后对账）
- 零停机执行步骤
- 故障回退预案

## Workflow

1. 设计前向迁移与回滚路径
2. 定义迁移前后数据校验规则
3. 制定低停机发布步骤
4. 执行演练并固化回退预案
