# perf-regression-guard

## Overview

把性能问题前置到发布前，避免“上线后才发现变慢”。

## When to Use

- 核心链路改动后
- 大版本发布前
- 容量或响应时间告警频繁

## Outputs

- 基线指标（延迟/吞吐/资源）
- 回归对比报告
- 慢点归因与优化建议
- 容量风险预警

## Workflow

1. 建立或更新性能基线
2. 对改动版本执行对比压测
3. 定位回归慢点与资源瓶颈
4. 输出优化与容量建议
