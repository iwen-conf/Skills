# Project Definition

这个技能用于帮助用户生成、梳理和完善一个清晰结构化的项目定义文档（Product Requirements Document / Project Blueprint）。

## 核心工作流

1. **理解需求**：当用户提供一个项目想法或初步构思时，分析其中的关键要素。如果信息不足，可以通过对话引导用户提供更多背景。
2. **使用模板**：所有的项目定义输出必须严格遵循 `references/template.md` 中的结构要求。不要随意增加或删除章节。
3. **输出要求**：
   - 语言要精炼、专业。
   - 每个小节的内容要直接、清晰，避免空话套话。
   - 在“核心概念”、“主要对象”和“核心功能”等部分，尽量使用列表的形式使其清晰易读。
   - 如果用户已经提供了部分信息，请自动填充到对应模块；对于缺失的部分，可以在输出的草案后询问用户，或者根据常识和经验给出建议。

## 参考资源

- **模板**：请查阅 [references/template.md](references/template.md) 了解项目定义文档的标准大纲和各项具体要求说明。在输出最终方案时，务必按照此文件的结构进行。
