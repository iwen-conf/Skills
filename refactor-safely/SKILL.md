# refactor-safely

## Overview

重构以行为不变为前提，强调可验证与可回退。

## When to Use

- 技术债清理与可维护性提升
- 模块解耦、职责重组
- 性能优化前的结构整理

## Outputs

- 重构范围与不变性声明
- 行为对照清单（Before/After）
- 回归测试点与风险点
- 分步提交建议

## Workflow

1. 定义重构边界与行为不变约束
2. 建立重构前后行为对照点
3. 执行小步重构与回归验证
4. 输出风险与回退建议
