# UML 图型目录与证据映射

## 图型到证据的映射

| 图型 | 英文标识 | 主要证据来源 | 常见输出价值 |
|---|---|---|---|
| 类图 | class | 领域模型、DTO、实体定义、接口定义 | 展示静态结构与继承/组合关系 |
| 对象图 | object | 运行时实例样本、测试夹具、示例数据 | 展示某一时刻对象实例关系 |
| 组件图 | component | 模块边界、服务目录、依赖声明 | 展示系统组件职责与依赖 |
| 部署图 | deployment | k8s/docker/terraform/ansible 配置 | 展示运行节点、部署拓扑与连接 |
| 包图 | package | 目录分层、包命名空间、import 关系 | 展示模块分组和耦合关系 |
| 复合结构图 | composite-structure | 核心组件内部结构、端口与连接器 | 展示组件内部协作结构 |
| 配置文件图 | configuration | `.env`、yaml/toml/json、配置中心 | 展示配置来源、覆盖顺序与依赖 |
| 用例图 | use-case | PRD、用户旅程、角色权限模型 | 展示角色与系统功能交互边界 |
| 活动图 | activity | 业务流程、编排逻辑、状态流转代码 | 展示流程步骤与分支并行 |
| 状态机图 | state-machine | 状态枚举、状态迁移规则、事件处理 | 展示对象生命周期与迁移条件 |
| 序列图 | sequence | API 调用链、事件时序、日志链路 | 展示消息时序与调用因果 |
| 通信图 | communication | 服务交互关系、协议与通道 | 展示对象间通信与连接关系 |
| 交互概述图 | interaction-overview | 跨场景交互编排、子序列组合 | 展示高层交互流程 |
| 时间图 | timing | 实时事件、超时重试、SLA/SLO 约束 | 展示时间约束与状态随时间变化 |

## 适用性判定建议

- `required`：该图直接支撑当前沟通目标（评审/交接/发布）。
- `recommended`：该图能显著降低理解成本，但非阻塞。
- `not-applicable`：项目阶段或证据不足，不适合当前产出。

## E-R 图（陈氏画法）

E-R 图是数据建模补充产物，不计入 UML 14 图型总数。  
当项目存在核心领域数据模型、复杂关系或数据库重构需求时，建议输出 `er-chen.mmd`。

| 图型 | 记法要求 | 最低交付 |
|---|---|---|
| E-R 图 | Mermaid 语法 + 陈氏画法（Chen Notation） | 实体、联系、关键属性、基数约束 |

详细规范见 `notation-standards.md`。

## 命名规范建议

- 文件名使用英文标识：`class.mmd`、`deployment.mmd`。
- 图标题可用中文业务语义，避免纯技术缩写。
- 每张图保留“证据来源”注释，便于后续更新。
